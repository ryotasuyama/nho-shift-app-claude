# 画面仕様書

## NHO 看護師シフト管理アプリ — MVP

---

## 1. ログイン画面（`/login`）

**利用者:** 全ユーザー

### 1.1 表示要素

| 要素 | 型 | 説明 |
|------|------|------|
| メールアドレス | テキスト入力 | 必須。メール形式バリデーション |
| パスワード | パスワード入力 | 必須。マスク表示（トグルで表示切替可） |
| ログインボタン | ボタン | 入力が空の場合は非活性 |
| エラーメッセージ領域 | テキスト | ログイン失敗時に赤色で表示 |

### 1.2 振る舞い

- ログイン成功時、`must_change_password = true` ならパスワード変更画面（`/settings/password`）へリダイレクト。
- ログイン成功時、ロールに応じて遷移先を分岐:
  - `admin` → `/dashboard`
  - `staff` → `/home`
- ログイン失敗時: 「メールアドレスまたはパスワードが正しくありません」と表示。

### 1.3 エラーメッセージ

| 条件 | メッセージ |
|------|---------|
| 未入力 | 「メールアドレスを入力してください」/「パスワードを入力してください」 |
| 形式不正 | 「メールアドレスの形式が正しくありません」 |
| 認証失敗 | 「メールアドレスまたはパスワードが正しくありません」 |
| レート制限 | 「ログイン試行回数が上限に達しました。しばらくしてから再試行してください」 |

## 2. 管理者ダッシュボード（`/dashboard`）

**利用者:** 管理者
**権限ガード:** `/admin/*` は未認証なら `/login` にリダイレクトし、認証済みだが `role != admin` の場合は `/home` にリダイレクト

### 2.1 表示要素

| 要素 | 説明 |
|------|------|
| 現在のターム情報 | 直近の `collecting` / `generating` / `adjusting` ステータスのターム |
| ターム進捗サマリー | 各タームのステータス一覧（直近3ターム） |
| 希望休申請状況 | 受付中タームの申請済み/未申請スタッフ数 |
| クイックアクション | 「シフト生成画面へ」「スタッフ管理へ」「ターム設定へ」のリンク |
| 有効スタッフ数 | チームA/B別の人数 |

## 6. 希望休入力画面（`/requests`）

**利用者:** スタッフ
**レスポンシブ:** スマートフォン対応

### 6.1 表示要素

| 要素 | 説明 |
|------|------|
| 対象ターム選択 | `collecting` ステータスのタームを表示 |
| カレンダーUI | ターム期間（28日間）のカレンダー。タップで日付選択 |
| 選択済み日付 | 選択中の希望休（最大3日）を一覧表示 |
| 残り申請可能数 | 「あと N 日選択できます」 |
| 申請ボタン | 1日以上選択で活性。押下時、未申請の選択日を1件ずつ `POST /requests` で順次送信する |
| 締切日 | `YYYY-MM-DD 23:59(JST)` として表示。締切後は「受付は終了しました」 |

### 6.2 状態別表示

| 状態 | 表示 |
|------|------|
| 受付中（`collecting` かつ締切前） | カレンダー選択可能。申請・変更・取消可能 |
| 受付終了（`collecting` だが締切後） | カレンダー表示のみ（選択不可）。申請済み内容の確認のみ。「申請受付は終了しました（締切日: YYYY-MM-DD）」と表示。ターム一覧APIは `collecting` として返すが、フロントエンドで `request_deadline` と現在日時を比較して締切後表示に切り替える |
| ターム未公開 | 「現在受付中のタームはありません」 |

### 6.3 エラーメッセージ

| 条件 | メッセージ |
|------|---------|
| 4日目選択 | 「希望休は最大3日までです」 |
| ターム期間外 | 「選択された日付はターム期間外です」 |
| 締切後の操作 | 「申請受付は終了しています」 |

## 7. シフト表画面 — 編集（`/admin/shifts/:termId`）

**利用者:** 管理者
**対応端末:** PC（1280×720以上）

### 7.1 グリッドレイアウト

| 要素 | 説明 |
|------|------|
| 行 | スタッフ（スタッフコード + 氏名）。チーム別にグループ化 |
| 列 | 日付（28日間）。曜日・祝日を表示 |
| セル | 勤務記号（日・準・深・休・代・希）をクリックで変更 |
| 右端列 | スタッフ別統計（勤務時間・夜勤回数・週休数） |
| 下端行 | 日別統計（日勤数・準夜数・深夜数） |

### 7.2 セルの背景色

| 条件 | 背景色 |
|------|--------|
| 土曜日 | 薄青 |
| 日曜日 | 薄赤 |
| 祝日 | 薄桃 |
| ハード制約違反 | 赤枠 |
| ソフト制約違反 | 黄枠 |
| 手動編集済み | 左上に小さなドット |

### 7.3 セル編集

- セルクリック → 勤務種別選択ポップオーバー（日 / 準 / 深 / 休 / 代）
- 選択後、即座に制約チェック実行
- 変更は Undo/Redo スタックに積まれる（未保存状態）
- `希` は希望休申請由来の表示であり、手動入力候補には含めない

### 7.4 ツールバー

| ボタン | 活性条件 | 説明 |
|--------|---------|------|
| 自動生成 | `collecting` 時（既存データがある場合は確認ダイアログ） | シフト自動生成を実行 |
| 再生成 | `adjusting` 時 | 別パターンで再生成。**再生成前に現在のデータがスナップショットとして自動保存される旨を確認ダイアログで表示** |
| 保存 | 未保存の変更がある場合 | 有効スタッフ分の current working copy を完全置換で永続化 + スナップショット作成 |
| Undo | Undo スタックが空でない場合 | 直前の編集を取り消し |
| Redo | Redo スタックが空でない場合 | 取り消した編集をやり直し |
| 確定 | `adjusting` 時 | シフトを確定・公開 |
| 差し戻し | `confirmed` 時 | `adjusting` に戻す |
| PDF出力 | `adjusting` or `confirmed` 時 | PDF ダウンロード |
| 履歴 | スナップショットが存在する場合 | 履歴一覧パネルを開く |

### 7.5 違反サマリーパネル

| 表示要素 | 説明 |
|---------|------|
| フェーズ1必須制約違反一覧 | 赤色。違反ID・対象スタッフ・違反内容 |
| 警告一覧 | 黄色。フェーズ2制約・ソフト制約を表示 |
| 違反数サマリー | 「必須: N件 / 警告: M件」 |
| 違反行クリック | 該当セルにスクロール＆ハイライト |

### 7.6 統計パネル

| 表示要素 | 説明 |
|---------|------|
| スタッフ別勤務時間 | 合計時間と個別標準時間（基準155h、代休分を控除）との差分 |
| スタッフ別夜勤回数 | 準夜回数・深夜回数 |
| スタッフ別週休回数 | 週休数・法定週休数 |
| 日別配置人数 | 各日の日勤・準夜・深夜の人数 |
| チーム別バランス | チームA/Bの配置比率 |

### 7.7 生成中のUI

- ローディングオーバーレイ表示
- 経過時間カウンター（「生成中... 15秒」）
- 生成中は他の編集操作・再生成・画面内主要アクションを非活性化する
- **キャンセル操作は提供しない**（生成完了 / タイムアウト / 失敗を待つ）
- 経過時間カウンターは**クライアント側のタイマー**で表示する（サーバーからのリアルタイム進捗通知は行わない。API は同期呼び出しで最大60秒レスポンスを待つ）
- 55秒経過時: 途中結果を表示し、「最終解ではありません」のメッセージを表示
- 生成失敗時: エラーダイアログ「シフト生成中にエラーが発生しました。」+ ステータスが `collecting` に戻る旨を表示
- **ネットワークタイムアウト時:** ブラウザ側で接続が切れた場合は、「生成処理の状態を確認できませんでした。画面を再読み込みしてください。」と表示し、再読み込みでタームのステータスを確認する

### 7.8 楽観的ロックエラー時のUI

- 保存時に `409 OPTIMISTIC_LOCK_ERROR` が返された場合:
  - ダイアログ「他の操作によりシフトデータが更新されました。最新データを読み込みますか？」
  - 「読み込む」ボタン → 最新データで画面をリフレッシュ（未保存の変更は破棄）
  - 「キャンセル」ボタン → ダイアログを閉じる（ユーザーが手動で対応）
- スナップショット復元後は、snapshot に保存された `is_manual_edit` に基づいて手動編集ドットを再描画する。`true` のセルのみドット表示、`false` は非表示
- **スタッフ減少時の表示:** スナップショット復元時に、保存時と比べて無効化されたスタッフがいた場合、「以下のスタッフは無効化されたため復元対象から除外されました: [スタッフ名一覧]」と情報メッセージを表示する。復元後に制約違反が発生した場合は通常通り違反サマリーパネルに表示される

### 7.9 離脱防止

- **ブラウザ離脱（タブ閉じる / リロード / ブラウザバック）:** 未保存の変更がある場合のみ `beforeunload` によるブラウザ標準の確認ダイアログを表示する
- **アプリ内遷移（サイドバー / Link / router.push）:** 未保存の変更がある場合のみ、アプリ独自の確認モーダルを表示する
- 保存後は `isDirty = false` となり、ブラウザ離脱・アプリ内遷移のいずれも確認なしで遷移可能

## 8. シフト表画面 — 閲覧（`/shifts/:termId`）

**利用者:** スタッフ
**権限:** `confirmed` タームのみ表示
**レスポンシブ:** スマートフォン対応

### 8.1 表示要素

| 要素 | 説明 |
|------|------|
| ターム期間 | ヘッダーに表示 |

## 9. パスワード変更画面（`/settings/password`）

**利用者:** 全ユーザー

### 9.1 表示要素

| フィールド | 型 | 必須 | バリデーション |
|------------|------|:--:|----------------|
| 現在のパスワード | パスワード | ○ | |
| 新しいパスワード | パスワード | ○ | 英数字混在8文字以上 |
| 新しいパスワード（確認） | パスワード | ○ | 上記と一致 |

### 9.2 エラーメッセージ

| 条件 | メッセージ |
|------|---------|
| 現在のパスワードが不正 | 「現在のパスワードが正しくありません」 |
| ポリシー不適合 | 「パスワードは英数字混在で8文字以上にしてください」 |
| 確認不一致 | 「新しいパスワードが一致しません」 |
| 変更成功 | 「パスワードを変更しました」（トースト通知） |
